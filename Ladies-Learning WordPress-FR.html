Qu'est-ce que WordPress?

  • Un système de gestion de contenu open source (code ouvert)
  • À la base, on l'utilisait beaucoup pour les blogues (et c'est toujours le cas!)
  • Maintenant il est assez puissant pour gérer des grands sites et applications (CNN, BBC, Mashable, etc...)
  • WordPress soutien plus de 60 millions de sites web à travers le monde.

WordPress.com VS WordPress.org

    WordPress.com est hébergé pour vous - pas de serveur ni de code requis !

    .com Ne permet pas de vraiment de personnalisation. Pas d'ajout de plugin ni de thèmes personnalisés.

    Nous utiliserons .org

    Il permet une personnalisation à 100% des thèmes et des extensions, oh-my!

Vocabulaire et "buzz words" - Bienvenue dans notre univers


localhost - Serveur local (MAMP ou WAMP) - Un programme qui roule sur votre ordinateur et qui imite les fonctionnalités d'un gros serveur payant. Vous pourrez travailler sur votre site web sans avoir à le téléverser nulle part.

Balises PHP - PHP est le langage sur lequel est basé WordPress. Quand nous devons l'utiliser, nous le mettons entre balises <?php and ?> pour que notre serveur l'identifie comme tel.

stylesheet/Feuille de style - Le fichier style.css de notre thème est l'endroit où est déterminé les couleurs, les polices et leurs tailles ainsi que le style de tous les éléments de votre site web.

Dashboard/tableau de bord - Le tableau de bord WordPress auquel vous accédez quand vous vous connectez pour la première fois à votre site web.

Codex - La documentation de WordPress. Très utile lorsque vous avez besoin d'aide ou d'une référence particulière: http://codex.wordpress.org/

Permalinks/Permaliens - URL humainement lisible. WordPress peut réécrire les liens comme MySite.com/?p=54 par MySite.com/contact-me

Thème - Une collection de fichiers que nous utilisons pour structurer et styliser notre site web. WordPress nous permet facilement de changer ou modifier un thème sans perdre le contenu de vos pages, ni de vos articles.

Template/Modèle - Un fichier spécifique dans notre thème qui détiens la structure totale d'une page ou d'une partie de page. Par exemple, page.php est le modèle dans notre thème qui défini la structure pour toutes nos pages.

Post/Article - Partie de contenu qui est conservée dans la base de données. Quand nous parlons d'un post, nous parlons d'un article de blogue (blog post).

Plugin/Extension (ou Plugiciel) - Un dossier à installer qui ajoute une fonctionnalité à votre site WordPress.

Query/Requête - Une requête à la base de données qui appelle certains articles selon certains paramètres.

Éditeur de texte - Un programme qui roule sur votre ordi utilisé pour écrire et éditer du code...

Tout code est écrit en format texte (plain text), donc vous pouvez juste utiliser notepad ou textedit pour modifier votre thème.

Utiliser un éditeur de texte rend l'écriture plus facile et nous vous recommandons d'en utiliser un aujourd'hui.

Il y a une tonne d'éditeurs de texte disponibles - Sublime Text, TextWrangler, NotePad++, Text Mate, vim...

Nous recommandons Sublime Text dont la période d'essai est illimitée mais vous êtes libre d'utiliser celui que vous voulez. http://www.sublimetext.com/2

N'essayez pas de modifier votre thème depuis l'administration.

Code + base de données = ♥

La base de données contient nos données. Le code définit comment et quand ces données apparaissent sur les pages.
Les deux travaillent ensemble pour produire du HTML.

/wp-admin et /wp-includes contiennent tout l'engin WordPress (on n'y touche pas).
/wp-content contient tous les contenus spécifiques comme le thème, les extensions et les images téléversées.

Anatomie d'un thème WordPress

Un thème WordPress est fait de nombreux fichiers modèles qui sont utilisés pour afficher différentes pages de votre site.


Par Exemple...

single.php est utilisé pour afficher un seul article (blog post) pendant que index.php est utilisé pour présenter la page principale du blogue où s'affiche la liste des articles récents. Plus sur ce sujet dans un instant...

Structure d'un thème de base

La plupart des pages peuvent être construite comme suit:

<?php get_header(); ?>

<?php get_sidebar(); ?>

CSS 101

Avant de personnaliser notre thème WordPress, il nous sera utile de connaître les bases du CSS. Nous n'approfondirons pas trop ce vaste sujet, mais jetons-y un oeil rapidement.

CSS se dit pour Cascading Style Sheets, qui permet d'ajouter du style aux éléments de notre site comme: l'entête, les images, les balises div (de contenu), les liens, les listes et plus encore.

Une déclaration CSS de base comprend 3 parties:

Un sélecteur - une propriété - une valeur

selector - property - value.

Les Sélecteurs (Selectors)

Le Sélecteur définit sur quel élément le CSS sera appliqué.

<div id="wrapper"> ... </div>
div#wrapper { ... }

<div class="post"> ... </div>
div.post { ... }

<span class="date"> ... </span>
span.date { ... }

<span class="category"></span> <div class="meta"></div>
span.category, div.meta { … }

<div class="parent"><div class="child"></div></div>
div.parent div.child { ... }

La Propriété et les Valeurs (Property and Value)

La propriété définit quel style est appliqué sur le sélecteur choisi et quelle valeur y sera attribuée.

Il existe des centaines de propriétés CSS, en voici quelques-unes très communes, appliquées dans ce cas-ci sur l'ID de wrapper (conteneur)

#wrapper {
  background: black;
  border: 2px solid green;
  font-family: Georgia, sans-serif;
}

Il y en a plusieurs sur votre feuille de référence, jetons-y un oeil maintenant...

La boucle

(The Loop)

Qu'est-ce qu'une boucle?

La boucle est l'essence d'un Thème WordPress. Elle sert à parcourir et à afficher chaque article qui sont disponibles sur une page donnée.

En clair, ce que dit la boucle:

« Tant qu'il y aura des articles à afficher, affiche-les. »

Notre page d'accueil du blogue montre 5 articles à la fois, mais une page d'article n'en montre qu'1 seul.

Chaque page requiert au moins une boucle de base afin d'afficher son contenu.

Pseudo Loop

La logique derrière la loop:

  si nous avons des articles
      tant que nous avons des articles
         1. Le Code ici est exécuté une fois pour chaque élément
de contenu
            extraire de la base de données
         2. Nous voulons afficher le titre, la date et le contenu
      fin tant que
  fin si

Vous êtes confuse?

Voyons comment WordPress fonctionne quand un internaute visite votre page d'accueil et lorsqu'il consulte un page de votre site.

Les étapes de la loop

1. La page du blogue est demandée par l'usager. WordPress interroge sa base de données pour tout le contenu de la page.

2. La base de données retourne 5 des plus récents articles.

3. WordPress prend le fichier header.php et débute la production de la page.

4. WordPress prend ensuite la loop. Comme nous avons 5 articles dans notre base de données, il reproduit alors le code de la loop 5 fois.

5. Une fois que WordPress a éxécuté sa loop, il continue pour afficher la sidebar et le footer.

Le Code

Un regard détaillé sur la production du titre et du contenu.

<?php if ( have_posts() ) while ( have_posts() ) : the_post(); ?>

    <h2><?php the_title(); ?></h2>
    <?php the_content(); ?>

<?php endwhile; ?>

Toujours confuse? Pas de soucis !

Notre thème WordPress comprend un loop.php qui s'occupe de la plupart des choses pour vous.

Vous devez simplement comprendre comment fonctionne la loop et à quoi elle ressemble pour que vous puissiez l'éditer vous-même !

Les balises du modèle (template tags)

Vous vous demandez comment afficher le contenu de votre page lors que vous êtes à l'intérieur de la loop?

Les balises du template sont de petites fonctions PHP qui produisent (output) le contenu de votre site.

Aucune connaissance en PHP n'est requise.

La plupart des template tags ne fonctionnent que dans la loop.

Les autres sont globales et peuvent être utilisées partout dans le site.

Vous vous souvenez d'avoir utilisé <?php echo home_url( '/' ); ?> dans le header ?

Exemple de template tags

Il existe des centaines de template tags

http://codex.wordpress.org/Template_Tags

Jetons un oeil aux plus utilisées.

<?php the_title(); ?> /*Affiche le titre*/

<?php the_content(); ?> /*Affiche le contenu*/

<?php the_excerpt(); ?> /*Affiche l'extrait*/

<?php the_permalink(); ?> /*Affiche l'URL de l'article'*/

<?php the_time(); ?> /*Affiche la date de l'article*/

<?php the_tags(); ?> /*Affiche les mots-clés*/

<?php the_category(); ?> /*Affiche les catégories*/

<?php comments_template(); ?> /*Affiche les commentaires*/

<?php the_author(); ?> /*Affiche l'auteur'*/

<?php the_post_thumbnail(); ?> /*Affiche l'image à la une*/

Étape 1: Depuis le tableau de bord, éditez votre page d'accueil. Écrivez quelques ligne à propos de vous et téléversez une photo.

Étape 2: Vous souvez-vous de notre page-home.php? Ouvrez-la et supprimez "Coming Soon..."

Étape 3: Nous allons maintenant écrire notre propre loop pour aller chercher et afficher le contenu de la page.

<?php if(have_posts()) while(have_posts()) : the_post(); ?>

<?php endwhile; ?>

Étape 4: Dans la loop, nous allons ajouter une div avec la classe homeBlurb et utiliser notre template tag <?php the_content(); ?> pour aller chercher le contenu de la page.

Notre loop ressemble maintenant à ceci:

<?php if(have_posts()) while(have_posts()) : the_post(); ?>
     <div class="homeBlurb"><?php the_content(); ?></div>

<?php endwhile; ?>

Maintenant que nous avons notre contenu de page, travaillons sur nos 3 colonnes.

Étape 5: Juste sous <?php endwhile; ?>, Créez une div vide avec la classe homeColumn.

<div class="homeColumn"></div>

Étape 6: dans cette div, insérez un <h3>Tag Cloud</h3> avec notre template tag <?php wp_tag_cloud(); ?>

<div class="homeColumn">
  <h3>Tag Cloud</h3>
  <?php wp_tag_cloud(); ?>
</div>

Étape 7: Créez une autre colonne avec la classe homeColumn, mais cette fois nous allons ajouter notre template tag pour aller chercher notre plus récent article.

Nous allons utiliser le marqueur <?php wp_get_archives(); ?>.

<div class="homeColumn">
  <h3>Latest Posts</h3>
  <ul><?php wp_get_archives('type=postbypost&limit=10'); ?></ul>
</div>

Nous encadrons notre template tag entre <ul> et </ul> puisqu'il s'agit d'une liste d'articles.

Étape 8: Créez la troisième et dernière colonne. Cette fois, nous utiliserons notre template tag pour aller chercher une liste de catégories.

Nous utiliserons le marqueur <?php wp_list_categories(); ?>.

<div class="homeColumn">
  <h3>Catégories</h3>
  <ul><?php wp_list_categories('title_li='); ?></ul>
</div>

Étape 9: On a fini !!! Nous vous avons fourni un minimum de style dans le fichier style.css. Trouvez la section intitulée "CUSTOM HOME PAGE STYLE" et amusez-vous à créer un look d'enfer pour votre site !

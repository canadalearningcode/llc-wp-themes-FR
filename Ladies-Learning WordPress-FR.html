Qu'est-ce que WordPress?

  • Un système de gestion de contenu open source(code ouvert)
  • À la base, on l'utilisait beaucoup pour démarrer des blogues (c'est toujours le cas!)
  • Maintenant qu'il est assez puissant pour gérer des grans sites et applications (CNN, BBC, Mashable, etc...)
  • WordPress soutient plus de 60 millions de sites web à travers le monde.

WordPress.com VS WordPress.org

    WordPress.com est hébergé pour vous - sans serveur ni code requis !

    .com Ne permet pas de vraiment de personnalisation. Ni d'ajout de plugin ou des thèmes personnalisés.

    Nous utiliserons .org

    Il permet une personnalisation à 100% des thèmes et des plugiciels, oh-my!

Vocabualaire et buzz words - Bienvenue dans l'univers


localhost - / Serveur local (MAMP ou WAMP) - Un programme qui roule sur votre ordi et qui imite les fonctionnalités d'un gros serveur payant. Vous pourrez travailler sur votre site web sans avoir à le téléverser nulle part.

Balises PHP - PHP est le langage du WordPress. Quand nous devons l'utiliser, nous le mettons entre balises <?php and ?> pour que notre serveur l'identifie comme tel.

stylesheet/ feuille de style - Le fichier style.css de notre thème est l'endroit où est déterminé les couleurs, les polices et leurs tailles ainsi que le style de tous les éléments de votre site web.

Dashboard/tableau de bord - Le tableau de bord WordPress auquel vous accédez quand vous vous connectez pour la première fois à votre site web.

Codex - La documentation WordPress. Très utile lorsque vous avez besoin d'aide ou d'une référence particulière: http://codex.wordpress.org/

Permalinks/Permaliens - URL humainement lisible. WordPress peut réécrire les liens comme MySite.com/?p=54 pour MySite.com/contact-me

Theme - Une collection de fichiers que nous utilisons pour structurer et styliser notre site web. WordPress nous permet facilement de changer ou modifier un thème sans perdre le contenu de vos pages, ni vos articles.

Template/Modèle -- Un fichier spécifique dans notre thème qui détiens la structure totale d'une page ou d'une partie de page. Par exemple, page.php est le modèle dans notre thème qui défini la structure pour toutes nos pages.

Post/Article - Partie de contenu qui est conservée dans la base de données. Quand nous parlons d'un post, nous parlons d'un article de blogue (blog post).

Plugin/ Extension (ou Plugiciel) - Un dossier à installer qui ajoute une fonctionnalité à votre site WordPress.

Query/ Requête - Une requête à la base de données qui appelle certains articles selons certains paramètres.

Text Editor/Éditeur de texte - Un programme qui roule sur votre ordi utilisé pour écrire et éditer du code...

Tout code est écrit en format texte (plain text), donc vous pouvez utiliser notepad ou textedit pour éditer votre thème.

Utiliser un éditeur de texte rend l'écriture plus facile et nous vous en recommanderons un aujourd'hui.

Il y a une tonne d'éditeurs de texte disponibles - Sublime Text, text wrangler, NotePad++, Text Mate, vim...

Nous recommandons Sublime Text dont la période d'essai est illimitée.

http://www.sublimetext.com/2

N'essayez pas de modifier votre thème depuis le tableau de bord WordPress !

CSS 101

Avant de personnaliser notre thème WordPress, il nous sera utile de connaître les bases du CSS. Nous n'approfondirons pas trop ce vaste sujet, mais jetons-y un oeil rapidement.

CSS tient pour Cascading Style Sheets, qui permet d'ajouter du style aux éléments de notre site comme: l'entête, les images, les balises div (de contenu), les liens, les listes et plus encore.

Une déclaration CSS de base comprend 3 parties:

Un sélecteur - une propriété - une valeur

selector - property - value.

Les Sélecteurs (Selectors)

Le Sélecteur définit sur quel élément le CSS sera appliqué.

<div id="wrapper"> ... </div>
div#wrapper { ... }

<div class="post"> ... </div>
div.post { ... }

<span class="date"> ... </span>
span.date { ... }

<span class="category"></span> <div class="meta"></div>
span.category, div.meta { … }

<div class="parent"><div class="child"></div></div>
div.parent div.child { ... }

La Propiété et les Valeurs (The Property and Value)

La property définit quel styles est appliqué sur le sélecteur choisir et quelle valeur y sera attribuée.

Il existe des centaines de propriétés CSS, en voici quelques-unes très communes, appliqués dans ce cas-ci sur l'ID de wrapper (conteneur)

#wrapper {
  background (arrière-plan): black;
  border (bordure):2px solid green;
  font-family (type de police):Georgia, sans-serif;
}

Il y en a plusieurs sur votre feuille de référence, jetons-y un oeil maintenant...

La boucle (The Loop)

Qu'est-ce que la boucle de programme?

La boucle est l'essence d'un Thème WordPress. Elle sert à itérer et à afficher chaque article rendu disponible sur une page.

La boule, en clair:

« Tant qu'il y aura des articles, affiche-les. »

Notre page d'accueil montre 5 blog post à la fois, mais une page d'article n'en montre qu'1 seul.

Chaque page requiert au moins une boucle de base afin d'afficher son contenu.

Pseudo Loop

La logique derrière la loop:

  if we have posts
      while we have posts
         1. The Code in here is run once for every piece of content
            retrieved from the database
         2. We might want to output the title, the date and the content
      end while
  end if

Vous êtes confuse?

Voyons comment WordPress fonctionne quand un internaute visite votre page d'accueil et lorsqu'il consulte un page de votre site.

Les étapes de la loop

1. La page du blogue est demandée par l'usager. WordPress interroge sa base de données pour tout le contenu de la page.

2. La base de données retourne 5 des plus récents posts.

3. WordPress prend le fichier header.php et débute la production de la page.

4. WordPress prend ensuite la loop. Comme nous avons 5 posts dans notre base de données, le reproduit alors le code de la loop 5 fois.

5. Une fois que WordPress a éxécuté sa loop, il continue pour afficher la sidebar et le footer.

Le Code

Un regard détaillé sur la production du titre et du contenu.

<?php if ( have_posts() ) while ( have_posts() ) : the_post(); ?>

    <h2><?php the_title(); ?></h2>
    <?php the_content(); ?>

<?php endwhile; ?>

Toujours confuse? Pas de soucis !

Notre thème WordPress comprend un loop.php qui s'occupe de la plupart des choses pour vous.

Vous devez simplement comprendre comment fonctionne la loop et à quoi elle ressemble pour que vous puissiez l'éditer vous-même !

Les balises du template

Vous vous demandez comment afficher le contenu de votre page lors que vous êtes à l'intérieur de la loop?

Les balises du template sont de petites fonctions PHP qui produisent (output) le contenu de votre site.

Aucune connaissance en PHP n'est requise.

La plupart des template tags sont utilisés dans la loop.

Les autres sont globales et peuvent être utilisées partout dans le site.

Vous vous souvenez d'avoir utilisé <?php echo home_url( '/' ); ?> dans le header ?

  <h3>Tag Cloud</h3>
  <?php wp_tag_cloud(); ?>
</div>

Étape 7: Créez une autre colonne avec la classe homeColumn, mais cette fois à l'intérieur nous allons ajouter notre template tag pour aller chercher notre plus récent article

Here we use the <?php wp_get_archives(); ?> template tag.

<div class="homeColumn">
  <h3>Latest Posts</h3>
  <ul>    <?php wp_get_archives('type=postbypost&limit=10'); ?>    </ul>
</div>

Nous englobons notre template tage entre <ul> et </ul> parce qu'il s'agit d'une liste d'articles.

Étape 8: Créons notre troisième et dernière colonne. Cette fois, nous utiliserons notre template tag pour aller chercher une liste de catégories.

Here we use the <?php wp_list_categories(); ?> template tag.

<div class="homeColumn">
  <h3>Catégories</h3>
  <ul>    <?php wp_list_categories('title_li='); ?>    </ul>
</div>

Étape 9: On a fini !!! Nous vous avons fournis un style bien minimal dans le fichier style.css. Trouvez la section intitulée "CUSTOM HOME PAGE STYLE" et amusez-vous à créer un look d'enfer pour votre site !
